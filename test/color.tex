\bigskip

In {\ActiveDVI}, colors can be specified either

\begin{itemize}
\item with names (taken from the {\LaTeX} package \verb"color.sty")
\item with RGB specifications (red, green, or blue color values, specified
as floating point numbers in the range 0.0 .. 1.0)
\item with CMYK specifications (cyan, magenta, yellow, or black color
values, specified as integers in the range 0 .. 255)
\end{itemize}

For instance, the background color of this page has been specified using the
\verb"yellow" predefined color of the \verb"color.sty" package, using:

\verb"\advibg{color=yellow}"
\advibg{color=yellow}

\bigskip

Note that using this kind of color specifications supposes that you
have load the \verb"color.sty" {\LaTeX} package (include the line

\verb"\usepackage{color}"

in the header of your document).

\section*{Named colors}

Colors can also be named using the keyword {\tt named}.
If you use named colors, the color names should be capitalized; for
instance:\\
\verb"\advibg{color=[named]White}"\\
or equivalently\\
\verb"\advibg{color=[named]{White}}"

\advibg{color=[named]{White}}

The names of available colors can be found in the {\tt dvipsnam.def}
file, generally at location\\
{\tt /usr/share/texmf/tex/latex/graphics/dvipsnam.def}

\subsection*{The CMYK specifications}

You may also explicitely use a CMYK specification:

\verb"\advibg{color=[cmyk]{1,0,1,0}}"

\bigskip

In this case, \verb"\advibg{color=[cmyk]1,0,1,0}" is not recognized,
since the comma is a field separator.

\advibg{color=[cmyk]{1,0,1,0}}

\newpage

\subsection*{The RGB specifications}

RGB specifications are similar to CMYK:

\verb"\advibg{color=[rgb]{1,0,1}}"

\advibg{color=[rgb]{1,0,1}}
